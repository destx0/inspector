## Design Context

### Users
Angular developers use the inspector locally while iterating on complex application flows. They need to save a working state and return to it without replaying earlier screens.

### Brand Personality
Compact, technical, unobtrusive.

### Aesthetic Direction
The checkpoint controls extend the existing dark inspector toolbar and use its dense, low-distraction developer-tool language.

### Design Principles
- Keep controls compact and immediately actionable.
- Preserve the inspector's existing dark visual system.
- Surface state and errors clearly without obscuring the inspected application.
- Make destructive actions secondary and explicit.
