## Design Context

### Users
Angular developers use the inspector locally while iterating on complex application flows. They need to save a working state and return to it without replaying earlier screens.

### Brand Personality
Compact, technical, unobtrusive.

### Aesthetic Direction
Raycast-inspired command palette as the single control surface: dark compact surfaces, quiet blue accent, keyboard-first chrome, minimal chrome. Checkpoints and inspector tools share one centered palette so the inspected application stays free of persistent floating controls.

### Design Principles
- Keep controls compact and immediately actionable.
- Prefer quiet glass surfaces and soft selection over heavy borders.
- Surface state and errors clearly without obscuring the inspected application.
- Make destructive actions secondary and explicit.
- Keyboard shortcuts are first-class UI, not afterthoughts.
